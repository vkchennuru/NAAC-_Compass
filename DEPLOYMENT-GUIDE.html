🚀 Deploy in 5 Steps
1
Create a Free GitHub Account
GitHub is a free platform for hosting websites and code. You only need to sign up once — it is completely free and takes 2 minutes.
Go to github.com/signup → Enter your email, create a password, choose a username.
💡 Tip for username: Use something like skr-college-kadapa or dr-krishnaveni — your URL will use this name.
→ Open GitHub Sign Up ↗
2
Create a New Repository Named "naac-compass"
A repository is like a folder on GitHub where your files live. Once created, it becomes your website's home.
After logging in to GitHub:
1. Click the + button (top right corner) → New repository
2. Repository name: naac-compass
3. Set visibility to Public (required for free hosting)
4. Tick the checkbox: ✅ Add a README file
5. Click Create repository
3
Upload Your Files
Now upload the 3 files from your computer into the GitHub repository.
Inside your new repository page:
1. Click Add file → Upload files
2. Drag these 3 files into the upload box:
   • index.html ← most important
   • README.md
   • DEPLOYMENT-GUIDE.html (optional)
3. Scroll down → click Commit changes
✅ Your files are now on GitHub!
4
Enable GitHub Pages (Free Hosting)
GitHub Pages turns your repository into a live website — completely free, no credit card, no expiry.
In your repository:
1. Click Settings (top menu bar)
2. In the left sidebar → click Pages
3. Under Branch → select main
4. Folder: leave as / (root)
5. Click Save
6. ⏳ Wait 1–2 minutes…
7. Refresh the page → you will see a green banner: "Your site is live at…"
5
Your Live URL — Share It!
Your NAAC Compass is now live on the internet, accessible by anyone with the link, on any device.
Your permanent URL will be: https://YOUR-USERNAME.github.io/naac-compass/ Replace YOUR-USERNAME with your actual GitHub username.

📱 Share this URL via WhatsApp, email, or on your college website.
🔄 To update the tool later: just upload a new index.html — the live site updates automatically within minutes.
🖥️ How to Use NAAC Compass
1
Open the URL in Chrome, Firefox, or any browser on mobile or desktop.
2
Select your role — IQAC Coordinator, Faculty, Principal, or Admin Staff.
3
Click "Enter NAAC Compass" to reach your personalised dashboard.
4
Click the 🧭 Setup Wizard banner on your first visit — it asks 7 quick questions and auto-generates tasks for your college.
5
Click any of the 10 attribute cards to see: what activities to perform, what documents to prepare, and how to report Input/Process/Output.
6
Use 📋 Tracker to add tasks, assign to faculty, set deadlines, and track completion.
7
Use 📅 Calendar to see what NAAC activities must be done each month of the year.
8
Use 📝 AQAR to fill your annual report — click 🤖 AI Draft on any field to auto-generate formal NAAC content.
9
Use 📁 DVV to tick off evidence documents and paste Google Drive links — exactly what NAAC verifiers request.
10
Use 📋 SSR to build your Self Study Report summary — click 🤖 AI Draft for the executive summary.
11
Use 📖 Glossary to look up any NAAC term in plain English — 40 terms with full forms and definitions.
12
Click the 🤖 floating button on any screen to chat with NAAC Buddy — ask anything in plain language.
13
Click 🏛️ College name button in the topbar to add profiles for multiple colleges — each with its own separate data.
14
All data saves automatically in your browser. It persists across sessions — no login needed.
15
Use EN / తె toggle in the topbar to switch between English and Telugu at any time.
💡 Important Tips

✅ Data is saved in your browser

All college data — compliance status, tasks, AQAR content, SSR data, DVV checklist — is stored in your browser's localStorage. It persists across sessions. Do not use Incognito/Private mode as data will be lost when you close the tab.

⚠️ AI features need internet

The NAAC Buddy chatbot and AI Draft buttons require an internet connection. All other features (attribute cards, tracker, calendar, glossary, DVV checklist) work without internet after the first load.

🔄 How to update the tool

  • Download the new index.html file when updates are available
  • Go to your GitHub repository → Add file → Upload files
  • Upload the new index.html → Commit changes
  • Your live site updates automatically within 2–5 minutes
  • All saved data in users' browsers is preserved across updates
